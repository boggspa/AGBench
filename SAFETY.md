# Safety Guidelines

This application wraps an AI agent capable of modifying local files and executing shell commands.

## Implemented Guardrails
- **Workspace Confinement**: Operations are only allowed within the explicitly selected workspace directory.
- **Approval Mode Constraints**: The app supports `plan`, `default`, and `auto_edit` modes. `auto_edit` is required for headless file edits, but it is NOT `yolo` mode. YOLO mode is strictly disabled and rejected by the UI.
- **Workspace Trust Assistant**: The app includes an embedded Trust Assistant in the Safety tab. It checks the official `~/.gemini/trustedFolders.json` file and provides an integrated terminal to run the official `/permissions trust` flow without leaving the app.
- **Session-only Trust**: For one-off tasks in untrusted folders, users can explicitly enable "Trust for this run only". This uses documented Gemini CLI mechanisms (`GEMINI_CLI_TRUST_WORKSPACE=true`) and does not write permanent trust to your system.
- **Model Capacity & Retries**: If the CLI encounters a 429 Model Capacity Exhausted error, the app tracks the retries. If a run completes successfully after transient errors, it is marked as "Success with warnings" to keep the user informed.
- **Log Redaction**: Before raw stdout/stderr logs are displayed in the "Raw Events" tab, they are redacted for email addresses, Bearer tokens, and local home paths. This tab also includes filters for easier auditing.

## Visual Redesign Safety
The visual/UX redesign did not introduce any safety regressions:
- All IPC channels remain narrow and validated.
- No new file-system or shell access was added to the renderer.
- The Diff Studio is read-only; it does not write or revert files.
- Native vibrancy is purely a visual surface treatment and does not affect sandboxing or `webPreferences`.
- The `contextIsolation: true`, `nodeIntegration: false`, and `sandbox: false` (for preload compatibility) configuration is preserved.

## No Branding/IP Copying
The visual system uses generic modern app patterns (glass sidebar, dark canvas, floating composer, inspector panel). It does not copy Codex, Windsurf, Claude, Gemini, Apple, Google, OpenAI, or Anthropic pixel-for-pixel. UI settings use generic names like "Soft Glass" and "Native Glass" rather than branded identities like "Liquid Glass."

## Manual Review
Always review the `git diff` generated by the agent before committing changes. Dangerous commands are not yet intercepted at the Electron layer, but the Gemini CLI handles its own approval flows.
